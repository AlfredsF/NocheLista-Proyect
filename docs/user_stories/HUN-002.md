# HUN-002 - Ver Catálogo

**Descripción:**  
Como cliente, quiero ver una lista de hoteles y habitaciones para elegir dónde hospedarme.

### Criterios de Aceptación
- [x] La base de datos en Supabase contiene al menos 3 hoteles de prueba y 2 habitaciones por hotel (*Seed data*).
- [x] Al entrar a la plataforma, el frontend en React consume el endpoint GET de Django y renderiza correctamente la cuadrícula con las tarjetas de los hoteles.
- [x] Al hacer clic en la tarjeta de un hotel, el usuario es redirigido a una vista detallada que muestra la información del hotel y sus habitaciones disponibles.
- [x] Si la base de datos está vacía, la interfaz debe mostrar un mensaje que indique "No hay establecimientos disponibles en este momento" en lugar de mostrar un error de carga.

### Definición de Hecho (DoD)
- [x] El código está subido al repositorio principal (Git) y fusionado en la rama de desarrollo (*Develop/Main*).
- [x] El código ha sido revisado y aprobado mediante un *Pull Request* por al menos un compañero del grupo.
- [x] No existen errores críticos ni advertencias (*warnings*) graves en la consola del navegador ni en la terminal de Django.
- [x] La interfaz en React es *responsive* (se ve y funciona correctamente tanto en monitores de PC como en pantallas de teléfonos móviles).
- [x] Cualquier cambio en la base de datos se ha reflejado correctamente mediante migraciones en Supabase.
