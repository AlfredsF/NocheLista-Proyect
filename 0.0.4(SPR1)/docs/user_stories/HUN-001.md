# HUN-001 - Sistema de Acceso

**Descripción:**  
Como usuario nuevo, quiero registrarme e iniciar sesión para acceder a NocheLista.

### Criterios de Aceptación

- [x] Un usuario nuevo puede registrarse llenando el formulario en React, y su información se guarda correctamente en la tabla de Supabase a través de la API de Django.
- [x] Un usuario registrado puede iniciar sesión y el sistema de Django le devuelve un token JWT válido.
- [x] El formulario en React valida que los campos no estén vacíos, que el correo tenga formato válido (@) y que la contraseña tenga una longitud mínima antes de enviar la petición.
- [x] Si el usuario ingresa credenciales incorrectas, la interfaz muestra un mensaje de error claro y amigable (ej. "Usuario o contraseña incorrectos").

### Definición de Hecho (DoD)

- [x] El código está subido al repositorio principal (Git) y fusionado en la rama de desarrollo (Develop/Main).
- [x] El código ha sido revisado y aprobado mediante un _Pull Request_ por al menos un compañero del grupo.
- [x] No existen errores críticos ni advertencias (_warnings_) graves en la consola del navegador ni en la terminal de Django.
- [x] La interfaz en React es _responsive_ (se ve y funciona correctamente tanto en monitores de PC como en pantallas de teléfonos móviles).
- [x] Cualquier cambio en la base de datos se ha reflejado correctamente mediante migraciones en Supabase.
